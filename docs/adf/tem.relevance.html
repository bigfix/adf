<static> defaultErrorHandler()

Default error handler for all relevance calls, unless overridden/disabled in the relevance call itself. It gets passed a relevance string and an Error object.

It can be customized, by default it logs an error if tem.config.isError is true and rethrows the error:

tem.relevance.defaultErrorHandler = function(relevance, error){
	if (tem.config.isError) {
		tem.logging.error('Error evaluating relevance: ' + error.message);
		tem.logging.error('     relevance: ' + relevance);
	}
	
	// rethrow error
	throw error;
}

<static> evaluate(relevance, callbackObj, customSynchronousErrorHandler) → {null|Object}

Used to evaluate session relevance.

Aliased as tem.evalRel

Note: Asynchronous relevance calls do not work in pre 8.1 Web Reports.

Sample Usage:

//Callback (asynchronous) and use tem.relevance.defaultErrorHandler
var onSuccessHandler = function(results) {
  alert(results);
}
tem.evalRel('now', {success:onSuccessHandler});	

//Callback (asynchronous) with a custom error handler
var onSuccessHandler = function(results) {
  alert(results);
}
var onErrorHandler = function (relevance, error) {
  alert('Error: ' + error.message + '    Relevance string: ' + relevance);
}
var results = tem.evalRel('nowa', {success:onSuccessHandler, failure:onErrorHandler});

//No callback (synchronous call) and use tem.relevance.defaultErrorHandler
var results = tem.evalRel('now');

//No callback (synchronous) with a custom error handler

var onErrorHandler = function (relevance, error) {
  alert('Error: ' + error.message + '    Relevance string: ' + relevance);
}
var results = tem.evalRel('nowa', null, onErrorHandler);

It is recommended that you generally call this function with asynchronous callbacks as if run asynchronously it will execute the relevance call in a separate thread than the UI runs in.

For plural properties this function returns a custom JavaScript type (RelevanceResult) that offers significant performance benefits, in particular with regards to sorting. RelevanceResult supports the following properties/methods:

  • length: Same as Array.length
  • []: Index operation, like array, but read only (assignment to elements is not supported). The result is the appropriate native JS type, or for tuples, a native array.
  • sort(): Like Array.sort(). Tuples are compared lexicographically.
  • sort(fn): where fn is a function: Like Array.sort(fn). The arguments passed to fn will be appropriate native JS types.
  • sort(i): Sort by the i'th tuple item. If that item is itself a tuple, a lexicographical comparison is used.
  • sort(i, fn): Sort by the i'th tuple item using the given comparison function, which will be passed two arguments of the appropriate native JS types.
  • toArray(): copy the result to a native JS array.
Parameters:
Name Type Argument Default Description
relevance String Relevance to execute
callbackObj Object If specified the call is made asynchronously. An object with a "success" property that defines a success callback which gets passed results when called and an optional "failure" property for a failure callback which gets passed the relevance string and error when called. If no failure property is defined, then tem.relevance.defaultErrorHandler will get called.
customSynchronousErrorHandler Boolean <optional>
 null For synchronous calls this handler will be used instead of tem.relevance.defaultErrorHandler.

<static> registerRelevance(relevance, callbackObj, id)

Added in IEM 9.0 (Gilman)
Not available in Web Reports

Registers a relevance query and a callback object (which contains a success function and a failure function) and a unique id such that the query and appropriate callback will run whenever the Dashboard Datastore (and other stores noted below) change. All subsequent calls to tem.relevance.registerRelevance using the same value for id will overwrite the original callback and relevance query.

Currently, the only way to un-register the callback is to call tem.relevance.registerRelevance with the same id and a relevance query of 'false'.

Sample Usage:

tem.relevance.registerRelevance('private variables test of current wizard',
	{	success:function(results){alert(results.length)},
		failure:function(errorStr){alert(errorStr)}
	},
	"uniqueID1"
	);

Note:The registered relevance query and callback get fired when ANY of the following gets updated:

  • ActionStore
  • ActionResultStore
  • AnalysisStore
  • CommentsStore
  • ComputerDataStore
  • ComputerGroupStore
  • DashboardDataStore
  • EffectivePermissionStore
  • FixletStore
  • FixletResultStore
  • FixletVisibilityStore
  • NamedFilterStore
  • RetrievedPropertyStore
  • PropertyResultStore
  • SiteStore
  • UnmanagedAssetStore
  • UserStore
  • UserRoleStore
  • LDAPDirectoryStore

Parameters:
Name Type Description
relevance String Relevance query you want to register.
callbackObj Object Defines a "success" and "failure" callback. Success gets passed results, failure gets passed an error string.
id String Unique identifier with which to register you query and callbacs. This parameter is likely to be removed in future version.
