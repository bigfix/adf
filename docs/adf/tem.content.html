Added in IEM 9.0 (Gilman)
Not available in Web Reports

Adds the file pointed to by filePath on the local console's filesystem to the custom site siteNameStr.

If siteNameStr is null or empty, the file is added to the master action site if the current console user is a master operator, or the user's operator site if the user is a non-master operator.

The file will be sent down to agent machines only if clientFile is set to true; otherwise it will be added to the site as a non-client file.

<static> importXML(xml, openDocuments, customSiteName, computerIDs) → {Array}

Not available in Web Reports Used to create content (Fixlet, Action, Analysis, Property, Baseline, ComputerGroup, Task).

This function is similar to tem.content.createContent except that it works with XML instead of model objects.

Note that you can construct your XML through string manipulation if you like, or you can use the XML Document Object Model provided by Microsoft via the HTML tag "xml" like this:

<xml id="myXMLDocument">
  <?xml version="1.0" encoding="UTF-8"?>
    <BES>
      <Fixlet>
        <Title>New Fixlet</Title>
          more XML here
      </Fixlet>
    </BES>
  </xml>

Then you can access the DOM using script like this:

 var doc = myXMLDocument.XMLDocument;
 var t = doc.selectSingleNode("BES/Fixlet/Title");
 t.text = "My Title";
 tem.content.importXML(doc.xml);

For more information on using the XML DOM, see Microsoft documentation.

If the import is successful, it returns an array of object IDs corresponding to and in the same order as the object elements in the XML. If the import is cancelled, it returns the JavaScript 'undefined' value. If something else goes wrong (for example, if the XML is invalid) a Javascript exception is thrown. Here's an example import: 

 function ImportAction(){
   try {
     result_ids = tem.content.importXML(ActionXML.xml, false);
   } catch (e) {
     alert('There was an error importing the action.');
     return false;
   }

   if (!result_ids) {
     alert('The action import was cancelled.');
   }								
 }
Parameters:
Name Type Argument Default Description
xml String
openDocuments Boolean <optional>
 true Specifies whether to open documents corresponding to newly imported items after they are imported.
customSiteName String <optional>
 TEM 8.1+ Only Used to import into a custom site, set this value to name of site. To import into your operator site, set customSiteName to undefined.
computerIDs Array <optional>
 TEM 8.1+ Only A list of computer IDs, so that for content where a Take Action Dialog can be applied at a later point, the user will only be able to target those computers if targeting by computer ID. Note that if targeting by name or property, other computers can also be targeted.

<static> openComputerGroup(cids, title)

Opens a computer group document in the console for an ad hoc set of computers, similar to how in the console you can select a set of computers in a computer list, right click, and select "View As Group" to open a computer group document that displays the selected set of computers as if they were a computer group.

Call the function with an array of computer ids, or a single computer id. Here are two examples:

Example 1:
tem.content.openComputerGroup(tem.evalRel("ids of bes computers"));

Example 2:
var ids = tem.evalRel("ids of bes computers");
var first5 = new Array(5);
for (var i = 0; i < 5; i++) {
  first5[i] = ids[i];
}

tem.content.openComputerGroup(first5);

The Console enables setting the computer group title, as follows:

tem.content.openComputerGroup(cids, "Computers needing patch X");

TEM enables tem.content.openComputerGroup() in Web Reports. However, when Web Reports aggregates multiple databases, this function will return every computer that matches any of the given ids across all databases. (For example, when calling tem.content.openComputerGroup( 1 ), if Web Reports is aggregating databases A and B, and both contain a computer with an id of "1", both computers will be returned as results.)

Parameters:
Name Type Argument Description
cids Number | String | Array Array of computer ids, or a single computer id. Strings convertible to numbers will be converted automatically.
title String <optional>
 Title that appears in document popup

<static> takeFixletAction(fixletID, siteID, contentID, parameters, callback)

Not available in Web Reports Used to bring up the take action dialog as if the user had clicked the take action link in a fixlet, it returns immediately and the take action dialog will be non-modal. The resulting action will be associated with the source Fixlet just as if the user clicked the take action link in the Fixlet.

You can optionally supply custom action parameters and a callback function that will be called after the action is created:

tem.content.takeFixletAction(tem.evalRel('id of current fixlet'),
                              tem.evalRel('id of current bes site'),
                              'Action1',
                              {'Parameter1': 'Value1', 'Parameter2': 'Value2'},
                              function(id){alert('Took an action with id ' + id);});
Parameters:
Name Type Argument Description
fixletID Number
siteID Number Note: The type of siteID is such that you can't pass the decimal version of the action site ID returned from Relevance, since the action site ID will be bigger than the largest signed integer. So in order to take an action on a custom fixlet, whose siteID is the action site ID: 
tem.content.takeFixletAction( fixletID, tem.evalRel('id of bes site whose (name of it is "ActionSite")')>>>0, contentID )
contentID String i.e. id of the type returned by the "content id of " inspector
parameters Object <optional>
 A JavaScript object whose property-value pairs will be used as action parameters. The user will be prompted for any parameters used in the script that are not specified in the object.

 A function taking one argument, the ID of the action that has been created. If the user cancels the Take Action dialog, the callback will not be called.

<static> takeSecureFixletAction(fixletID, siteID, contentID, parameters, secureParameters, callback)

Added in IEM 9.0 (Gilman)
Not available in Web Reports

Similar to tem.content.takeFixletAction, and allows adding secure parameters. Using secure parameters forces the action to be targeted statically, and then added to the mailboxes of any targeted agents. The parameters are encrypted so that only the targeted agents will be able to access them. They can then be used in the action script just like normal parameters

                                 {'Parameter1': 'Value1', 'Parameter2': 'Value2'},
