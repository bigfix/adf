HasStartTime Boolean
StartDateTimeOffset String Use StartDateTimeOffset to specify the start time using GMT as the reference time. Instead of setting StartDateTimeOffset as of TEM 8.1 patch #2+ you can specify the {String} StartDateTimeLocalOffset property to specify the start time using local time as the reference time.

For example, suppose it's January 1, 12:00 local time and your time zone is GMT-8.
  "StartDateTimeOffset = 1 day, 2 hours" sets the start time to Janary 2, 22:00.
  "StartDateTimeLocalOffset = 1 day, 2 hours" sets the start time to January 2, 14:00.

These values need to be set according to the following RegExp specified in the XML schema:

P([0-9]+D)?(T([0-9]+H)?([0-9]+M)?([0-9]+(\.[0-9]{1,6})?S)?)?

A sample value would be "P1DT23H59M48S"

fromXML

This method deserializes an XML string and populates the model object's properties based on the contents of the XML, recursively creating "sub-model objects" where needed.

For example:

var testFixlet = new tem.model.Fixlet();
testFixlet.Title = "Test Fixlet";
...
var testAction = new tem.model.FixletAction("action1");
testAction.ActionScript = "// do nothing";
testFixlet.addAction(testAction);

var fixletId = tem.content.createContent(testFixlet, true, false);
var fixletXML = tem.evalRel('(fixlet whose (id of it = ' + fixletId +') of bes sites whose (name of it = "ActionSite")) as xml');	
var builtFixlet = new tem.model.Fixlet();
builtFixlet.fromXML(fixletXML);

would result in the builtFixlet model object having all of its properties set based on the retrieved XML, including having its Action property populated with a tem.model.FixletAction object.

Parameters:
Name Type Description
nodeXML String String representation of an XML object.
